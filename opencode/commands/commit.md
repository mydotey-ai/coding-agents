description: 自动分析改动并生成 commit message 后提交
---

请直接完成本次提交，不要推送。

要求：
- 先切到 git 仓库根目录语义下工作，确保不要遗漏子目录外的改动
- 检查当前工作区和暂存区状态；如果没有可提交的改动，明确告知“没有需要提交的改动”并结束
- 分析未暂存改动、已暂存改动，以及最近 5 条提交信息，学习当前仓库的 commit 风格
- 基于改动内容生成简洁自然的 commit message，聚焦改动目的和影响，语言跟随仓库现有风格
- 将相关文件加入暂存区，避免把明显的敏感文件或无关文件提交进去
- 创建一次正常 commit，不要 push
- 如果 pre-commit hook 失败，先修复再重新提交
- 如果存在 merge conflict，停止并提示用户先解决冲突

完成后汇报：
- commit message
- commit SHA
- 当前分支名
