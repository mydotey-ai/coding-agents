# 文档规则

## 目录结构

文档统一放置在 `/docs/` 目录下，按类型分目录：

| 目录 | 用途 |
|------|------|
| `architecture/` | 架构设计文档 |
| `api/` | API 接口文档 |
| `database/` | 数据库设计文档 |
| `design/` | 设计系统和 UI 规范 |
| `development/` | 开发指南和规范 |
| `deployment/` | 部署文档 |
| `decisions/` | 架构决策记录 (ADR) |
| `plans/` | 开发计划 |
| `plans/phases/` | 各阶段实施文档 (phase-X-xxx.md) |

## 索引文件

每个目录通过 `README.md` 作为索引，只登记该目录下的直接文件和子目录，不递归展开子目录内容。新增或删除文档时同步更新所在目录的索引。

## 文档格式

- 使用 Markdown 格式，文件名采用 `kebab-case`，不加日期
- 索引文件中不包含文档版本信息
- 每个文档以一级标题开头，简要说明文档目的
- 章节使用二级、三级标题层级递进，不要跳级
- 代码块标注语言类型（`java`、`sql`、`bash` 等）
- 表格用于结构化数据（配置项、API 参数、状态对比等）

## 文档更新机制

- **新增功能**：同步更新对应类型的文档，并在所在目录 `README.md` 中登记
- **Phase 完成**：更新 Phase 文档和所在目录索引
- **架构变更**：更新相关架构文档，重大决策记录到 `/docs/decisions/adr.md`
- **API 变更**：更新 `/docs/api/` 下对应文档
