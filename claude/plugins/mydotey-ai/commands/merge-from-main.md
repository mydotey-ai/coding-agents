# merge-from-main

将本地 main 分支的代码合并到当前 worktree 分支。用于同步主分支的最新改动。

## 执行步骤

1. **切换到项目根目录**
   - 运行 `git rev-parse --show-toplevel` 获取 git 根目录路径
   - 如果当前目录不是根目录，切换到根目录

2. **检测当前状态**
   - 运行 `git worktree list` 检查当前是否在 worktree 中，如果不在则提示用户此命令仅用于 worktree
   - 运行 `git branch --show-current` 获取当前分支名
   - 如果当前是 main 或 master，提示用户此命令仅用于非主分支
   - 运行 `git status` 检查是否有未提交的改动，如有则提示用户选择：先提交或放弃操作

3. **确定主分支名称**
   - 运行 `git branch | grep -E 'main|master'` 检查存在哪个主分支
   - 优先选择 main，如果没有则选择 master

4. **执行合并**
   - 运行 `git merge <main-branch>` 将主分支的改动合并到当前分支
   - 如果有冲突，自动分析冲突内容并尝试解决
   - 如果冲突无法自动解决，与用户对话协商解决方案

5. **显示结果**
   - 显示合并完成状态

## 错误处理

- 当前已是主分支：提示用户切换到开发分支后再执行
- 有未提交改动：提示用户选择先提交或放弃操作
- 合并冲突：优先自动解决，无法解决时与用户对话处理

## 注意事项

- 此命令只在本地操作，不自动推送
- 在 worktree 中可以直接合并本地主分支，无需切换到主仓库
